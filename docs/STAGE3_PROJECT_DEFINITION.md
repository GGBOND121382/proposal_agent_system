# 阶段3：项目定义冻结

## 目标

阶段3只冻结项目研究对象、唯一中心命题、研究差距、研究问题、目标、研究内容、方法假设和验证边界。该阶段不生成申请书正文，不确认最终创新点，也不放行章节规划。

## 输入边界

阶段3只能使用阶段1设计输入与阶段2事实底座中已授权的事实。若阶段2缺少项目定义所需的直接事实，工作流必须先执行定向事实补充，并重新经过确定性校验、独立审查和人工确认。

必须具有直接事实绑定的对象包括：核心概念、问题陈述、当前差距、中心命题、研究属性、成熟度目标、正文目标页数和参考文献页数规则。

## 模型链路

1. 项目定义生成器输出完整结构化候选。
2. 确定性质量门检查Schema、上游哈希、ID闭环、事实权限、开放事项、可证伪性和阶段边界。
3. 首次生成若仅存在明确的格式或关系错误，系统保留原始响应并允许一次定向修复。
4. 修复后的完整候选重新执行全部确定性检查。
5. 独立Critic检查研究逻辑、工程载体边界和创新状态。
6. 人工Gate只确认本阶段允许冻结的对象。

## 恢复约束

- 定向修复最多一次。
- 修复请求必须携带原候选和确定性Finding。
- 修复不得改变已经通过校验的研究问题、中心命题、事实权限、开放事项和阶段边界。
- 原始生成响应、修复请求和修复响应必须同时保留在Trace中。
- 修复成功不能跳过独立Critic和人工Gate。

## 放行结果

阶段3通过后只允许进入论证架构阶段。正式模板、相关工作与比较基线、最终创新点、研究基础证据、团队、经费、周期和指标阈值继续保持未决。
