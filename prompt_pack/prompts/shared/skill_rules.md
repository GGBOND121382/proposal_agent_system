# 可调用技能与弱模型适配规则

运行时会在模型输出之后调用确定性技能。模型不需要直接操作文件、浏览器、图片或网络。

## Mermaid图形技能

当章节需要流程图、逻辑结构图、技术路线图、执行流或关系图时，只生成一个简单的Mermaid源码段落，不生成图片二进制，不输出Base64，不描述绘图软件操作。

段落文本格式必须为：

[[MERMAID]]图题|宽度厘米
flowchart LR
A[短标签] --> B[短标签]

规则：

- 优先使用 `flowchart LR` 或 `flowchart TB`；
- 单图建议不超过15个节点、25条边；复杂关系拆为多张图；
- 节点标签使用简短中文，不使用长段落；
- 不使用 `click`、JavaScript、外部链接、初始化指令或自定义HTML；
- Mermaid源码将由代码进行语法与内容校验，保存为 `.mmd`，再渲染为SVG和PNG；
- 图示段落必须绑定 `primary_claim_id`、`evidence_ids`、`novel_content_key` 和 `section_contract_id`，说明该图服务于哪项论证；
- 模型不具备绘图能力时仍可输出上述纯文本语句；只有已经通过蓝图审查的图示意图才允许使用可编辑模板回退；
- 章节标题本身不能触发自动插图，禁止为了图表数量补充装饰性流程图。

## 公共研究技能

公开研究模型只负责：提出检索问题、根据已归档的公开资料进行比较与综合、绑定Source Ref。搜索、网页抓取、PDF抽取、快照归档、Hash计算和来源清单由代码技能完成。

- 不得引用未出现在 `retrieved_sources` 或 `extracted_passages` 中的资料；
- 不得凭模型记忆补造作者、题名、年份、标准编号、URL或DOI；
- 每个Public Claim必须绑定实际归档来源；
- 资料真实性由归档目录中的原始快照、提取文本、元数据、访问时间和SHA-256校验。

## 弱模型任务边界

- 一次调用只完成当前Prompt定义的一种职责；
- 规划、正文生成、审查、检索、图形渲染和导出相互分离；
- 优先产生短而结构化的JSON字段，不在一个字段中混合多种任务；
- 无法完成时返回明确的缺失字段或Finding，不用泛化文字掩盖失败。
